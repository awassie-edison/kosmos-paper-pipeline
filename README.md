# Kosmos Paper Pipeline

Automated pipeline for discovering, evaluating, and downloading computational biology papers with public datasets for the Kosmos AI benchmarking system.

## What it does

1. **Search** — Queries Europe PMC and PubMed for recent open-access computational biology papers
2. **Filter** — Filters by journal tier, article type, and open-access status
3. **Evaluate** — Uses Claude API to assess each paper's hypothesis chain, dataset reproducibility, and benchmark suitability
4. **Verify sizes** — Checks actual dataset sizes via GEO FTP, Zenodo API, OpenNeuro, etc.
5. **Score & rank** — Applies composite scoring with topic diversity adjustment
6. **Download PDFs** — Fetches paper PDFs from Europe PMC, Unpaywall, or publisher sites
7. **Download datasets** — Downloads primary (and key reanalyzed) datasets from GEO, Zenodo, OpenNeuro, OMIX/HRA, Figshare, Dryad
8. **Upload to Edison** — Uploads datasets and generates `run.yaml` probe directories

## Setup

```bash
uv sync
export ANTHROPIC_API_KEY=sk-...
```

## Usage

```bash
# Full run with review gate (pauses before downloading)
python -m kosmos_pipeline -o ./runs/2026-04-01 --review --skip-upload

# Search + evaluate only (no downloads)
python -m kosmos_pipeline -o ./runs/2026-04-01 --skip-download --skip-upload

# Exclude already-benchmarked papers
python -m kosmos_pipeline -o ./runs/2026-04-01 \
  --exclude-dois existing_papers.xlsx --review

# Resume after interruption (checkpoints are automatic)
python -m kosmos_pipeline -o ./runs/2026-04-01

# Full pipeline including Edison upload
python -m kosmos_pipeline -o ./runs/2026-04-01 --stage DEV
```

## CLI options

| Flag | Description |
|---|---|
| `-o, --output-dir` | Output directory (default: `./runs/YYYY-MM-DD`) |
| `--review` | Pause after paper selection for review before downloading |
| `--skip-download` | Skip dataset download step |
| `--skip-upload` | Skip Edison upload step |
| `--max-candidates N` | Max papers to evaluate via Claude API (default: 150) |
| `--max-papers N` | Max papers in final output (default: 20) |
| `--model` | Claude model for evaluation (default: `claude-opus-4-6`) |
| `--exclude-dois FILE` | File with DOIs to exclude (one per line, or .xlsx) |
| `--stage` | Edison environment: DEV, STAGING, PROD |
| `--kosmos-opt-dir` | Path to kosmos-opt repo for Edison uploads |
| `--no-resume` | Start fresh, ignoring checkpoints |
| `-v, --verbose` | Debug logging |

## Output structure

```
runs/2026-04-01/
├── status.json                                # Live progress tracker
├── computational_biology_papers_2026.json     # Final curated paper list
├── manifest.json                              # Links paper → PDF → probe → data_entry
├── intermediate/                              # Checkpoints (for resume)
├── pdfs/                                      # Paper PDFs
├── datasets/                                  # Downloaded datasets
└── probes/                                    # Edison run.yaml files
```

## Dataset role classification

Each dataset accession is classified as:
- **`primary`** — Generated by the paper, always downloaded
- **`reanalyzed_key`** — From another study but essential to the paper's main results, downloaded
- **`reanalyzed`** — Used for minor comparison/validation only, not downloaded

## Supported repositories

GEO, Zenodo, OpenNeuro, OMIX/HRA (NGDC/CNCB), Figshare, Dryad, SRA
